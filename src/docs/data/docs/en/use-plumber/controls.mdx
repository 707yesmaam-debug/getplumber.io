Plumber checks your CI/CD configuration and project settings against a catalog of controls. The catalog is split per provider: GitLab controls inspect `.gitlab-ci.yml` and GitLab project settings, GitHub controls inspect `.github/workflows/*.yml` and GitHub repository settings. Pick the provider tab below to see the relevant catalog.

When a control is not respected, an issue is created. Click the `ISSUE-XXX` link in any row to see the full description, impact, and remediation for that issue.

# ★ Currently shipping

The GitHub controls below are enabled by default in the embedded `.plumber.yaml`. Their findings appear in the terminal output, the JSON report, the PBOM, and the pull-request comment.

<div class="not-content my-4 rounded-md border border-border bg-muted/30 px-3 py-2.5 text-[13px] leading-snug text-muted-foreground">
  <p class="mb-2 font-medium text-foreground">What Plumber scans (GitHub CLI)</p>
  <ul class="m-0 list-disc space-y-1 pl-4">
    <li><strong class="text-foreground">Source:</strong> committed files under <code style={{fontSize: '0.85em'}}>.github/workflows/</code> in the analyzed ref (local clone or remote fetch).</li>
    <li><strong class="text-foreground">Static YAML only:</strong> no org/repo Variables, no runtime <code style={{fontSize: '0.85em'}}>$GITHUB_ENV</code> writes, no evaluation of GitHub expression syntax in values.</li>
    <li><strong class="text-foreground">Step <code style={{fontSize: '0.85em'}}>uses:</code></strong> third-party <code style={{fontSize: '0.85em'}}>owner/repo@ref</code> actions (ISSUE-108/114). Not local <code style={{fontSize: '0.85em'}}>./.github/actions/*</code>, not job-level reusable-workflow <code style={{fontSize: '0.85em'}}>uses:</code>, not nested actions inside composites.</li>
    <li><strong class="text-foreground">API-backed rules (ISSUE-108, ISSUE-114):</strong> need <code style={{fontSize: '0.85em'}}>gh auth login</code> or <code style={{fontSize: '0.85em'}}>GH_TOKEN</code>. Without auth they <strong class="text-foreground">abstain</strong> (no finding, not a pass).</li>
  </ul>

## GitHub Actions — Supply chain
| Scope | Control | What it checks | Why it matters |
|-------|---------|----------------|---------------|
| <ProductScopeBadge scope="cli" /> | Third-party actions must be pinned by commit SHA [ISSUE-104](/docs/use-plumber/issues/ISSUE-104?p=github) | <span style={{fontSize: '0.85em'}}>Verifies every <code style={{fontSize: '0.85em'}}>uses: owner/repo@ref</code> is pinned by a 40-character commit SHA, not a mutable tag or branch.</span> | <span style={{fontSize: '0.85em'}}>Defends against the March 2025 tj-actions / reviewdog supply chain compromise pattern (CVE-2025-30066), where attackers retag actions to point at malicious code that then runs with the caller's secrets.</span> |
| <ProductScopeBadge scope="cli" /> | Actions must not reference archived repositories [ISSUE-108](/docs/use-plumber/issues/ISSUE-108?p=github) | <span style={{fontSize: '0.85em'}}>Step-level <code style={{fontSize: '0.85em'}}>uses: owner/repo@ref</code> in committed workflows. Queries GitHub for <code style={{fontSize: '0.85em'}}>archived: true</code> (one cached API call per <code style={{fontSize: '0.85em'}}>owner/repo</code>). Requires <code style={{fontSize: '0.85em'}}>gh</code> / <code style={{fontSize: '0.85em'}}>GH_TOKEN</code>; abstains without auth.</span> | <span style={{fontSize: '0.85em'}}>Archived repos stop receiving patches. Does not inspect reusable-workflow callees, local actions, or runtime installs. PBOM: <code style={{fontSize: '0.85em'}}>archived: true</code>.</span> |
| <ProductScopeBadge scope="cli" /> | Actions must not carry known CVEs [ISSUE-114](/docs/use-plumber/issues/ISSUE-114?p=github) | <span style={{fontSize: '0.85em'}}>Same step-level <code style={{fontSize: '0.85em'}}>uses:</code> scope as ISSUE-108. Queries GitHub Advisory Database (<code style={{fontSize: '0.85em'}}>actions</code> ecosystem); semver-filters by <code style={{fontSize: '0.85em'}}>vulnerable_version_range</code> when the ref is a tag. SHA pins without a resolvable tag may match conservatively.</span> | <span style={{fontSize: '0.85em'}}>Catches known-vulnerable action versions (e.g. tj-actions CVE-2025-30066). Requires API auth; abstains without it. PBOM: <code style={{fontSize: '0.85em'}}>hasCve</code> + <code style={{fontSize: '0.85em'}}>advisories: [GHSA-…]</code>.</span> |

Also <code style={{fontSize: '0.85em'}}>run:</code> lines that write forbidden names to <code style={{fontSize: '0.85em'}}>$GITHUB_ENV</code>.</span> | <span style={{fontSize: '0.85em'}}>Debug toggles dump masked secrets into logs. Org/repo Variables with no YAML reference and UI "Re-run with debug logging" remain out of scope.</span> |
