Plumber allows defining controls for GitLab projects, covering both CI/CD configuration and project settings. When a control is not respected, an issue is created.

**Scope** labels show where the control applies: **All** means Plumber Platform and the Open Source CLI; **CLI** means the Open Source CLI reports this finding only (it is not enforced as a Platform control).

## CI/CD Container Images
| Scope | Control | What it checks | Why it matters |
|-------|---------|----------------|---------------|
| <ProductScopeBadge scope="all" /> | Containers images must come from authorized sources [ISSUE-101](/docs/use-plumber/issues/ISSUE-101) | <span style={{fontSize: '0.85em'}}>Verifies that container images used to run your CI/CD pipelines come from authorized and trusted sources.</span> | <span style={{fontSize: '0.85em'}}>Helps mitigate security risks introduced by the use of malicious, compromised, or vulnerable images.</span> |
| <ProductScopeBadge scope="all" /> | Container images must not use forbidden tags [ISSUE-102](/docs/use-plumber/issues/ISSUE-102) | <span style={{fontSize: '0.85em'}}>Verifies that container images used to run your CI/CD pipelines rely on authorized tags.</span> | <span style={{fontSize: '0.85em'}}>Helps mitigate both security and functional risks introduced by the use of unverified, outdated, or compromised image versions.</span> |
| <ProductScopeBadge scope="cli" /> | Container images must be pinned by digest [ISSUE-103](/docs/use-plumber/issues/ISSUE-103) | <span style={{fontSize: '0.85em'}}>Verifies that container images are referenced by their SHA256 digest rather than a mutable tag.</span> | <span style={{fontSize: '0.85em'}}>Prevents supply chain attacks where a tag is overwritten with a compromised image, guaranteeing exact image content in every pipeline run.</span> |

## CI/CD Variables
| Scope | Control | What it checks | Why it matters |
|-------|---------|----------------|---------------|
| <ProductScopeBadge scope="all" /> | CI/CD variables must be protected [ISSUE-201](/docs/use-plumber/issues/ISSUE-201) | <span style={{fontSize: '0.85em'}}>Verifies that CI/CD variables used in a project have the protected field enabled.</span> | <span style={{fontSize: '0.85em'}}>Ensures sensitive values are restricted to protected branches or tags, reducing unauthorized exposure.</span> |
| <ProductScopeBadge scope="all" /> | CI/CD variables must be masked [ISSUE-202](/docs/use-plumber/issues/ISSUE-202) | <span style={{fontSize: '0.85em'}}>Verifies that CI/CD variables used in a project have the masked field enabled.</span> | <span style={{fontSize: '0.85em'}}>Prevents variable values from being exposed in pipeline logs, reducing the risk of leaks.</span> |
| <ProductScopeBadge scope="cli" /> | Pipeline must not enable debug trace [ISSUE-203](/docs/use-plumber/issues/ISSUE-203) | <span style={{fontSize: '0.85em'}}>Verifies that <code style={{fontSize: '0.85em'}}>CI_DEBUG_TRACE</code> and <code style={{fontSize: '0.85em'}}>CI_DEBUG_SERVICES</code> are not enabled in the pipeline configuration.</span> | <span style={{fontSize: '0.85em'}}>Prevents exposure of all CI/CD variable values in job logs, including secrets and tokens.</span> |
| <ProductScopeBadge scope="cli" /> | Pipeline must not use unsafe variable expansion [ISSUE-204](/docs/use-plumber/issues/ISSUE-204) | <span style={{fontSize: '0.85em'}}>Detects user-controlled CI variables expanded in shell re-interpretation contexts (<code style={{fontSize: '0.85em'}}>eval</code>, <code style={{fontSize: '0.85em'}}>sh -c</code>, <code style={{fontSize: '0.85em'}}>bash -c</code>, <code style={{fontSize: '0.85em'}}>source</code>).</span> | <span style={{fontSize: '0.85em'}}>Prevents command injection via crafted branch names, MR titles, or commit messages (OWASP CICD-SEC-1).</span> |

| <ProductScopeBadge scope="all" /> | Pipeline must not contain hardcoded jobs [ISSUE-401](/docs/use-plumber/issues/ISSUE-401) | <span style={{fontSize: '0.85em'}}>Verifies that no hardcoded job is used in CI/CD pipelines.</span> | <span style={{fontSize: '0.85em'}}>Improves maintainability and ensures compliance with best practices.</span> |
| <ProductScopeBadge scope="all" /> | Pipeline must not override template jobs [ISSUE-402](/docs/use-plumber/issues/ISSUE-402) | <span style={{fontSize: '0.85em'}}>Verifies that no job coming from an included template is overridden in the project CI/CD configuration.</span> | <span style={{fontSize: '0.85em'}}>Prevents inconsistent or insecure pipeline configurations caused by job overrides.</span> |

| <ProductScopeBadge scope="cli" /> | Security jobs must not be weakened [ISSUE-410](/docs/use-plumber/issues/ISSUE-410) | <span style={{fontSize: '0.85em'}}>Detects security scanning jobs (SAST, Secret Detection, Container Scanning, etc.) weakened by <code style={{fontSize: '0.85em'}}>allow_failure: true</code>, <code style={{fontSize: '0.85em'}}>rules:</code> overrides with <code style={{fontSize: '0.85em'}}>when: never</code> / <code style={{fontSize: '0.85em'}}>when: manual</code>, or <code style={{fontSize: '0.85em'}}>when: manual</code> at job level.</span> | <span style={{fontSize: '0.85em'}}>Prevents silently neutralized security scans that give a false sense of compliance (OWASP CICD-SEC-4).</span> |
| <ProductScopeBadge scope="cli" /> | Pipeline must not execute unverified scripts [ISSUE-411](/docs/use-plumber/issues/ISSUE-411) | <span style={{fontSize: '0.85em'}}>Detects jobs that download and immediately execute scripts from the internet (<code style={{fontSize: '0.85em'}}>curl &#124; bash</code>, <code style={{fontSize: '0.85em'}}>wget &#124; sh</code>, download-then-execute) without integrity verification.</span> | <span style={{fontSize: '0.85em'}}>Prevents supply chain attacks where a compromised URL serves a modified script that exfiltrates secrets (OWASP CICD-SEC-3, CICD-SEC-8).</span> |
